Architecture Patterns

Simple (2 Agents)

┌─────────────┐ ┌─────────────┐ │ Researcher │ ──→ │ Executor │ │ (Analysis) │ │ (Trading) │ └─────────────┘ └─────────────┘

Researcher analyzes markets, Executor handles trades. Good for single-strategy setups.

Standard (4 Agents)

┌─────────────┐ ┌─────────────┐ │ Researcher │ ──→ │ Analyst │ └─────────────┘ └──────┬──────┘ ↓ ┌─────────────┐ ┌─────────────┐ │Risk Manager│ ──→ │ Executor │ └─────────────┘ └─────────────┘

Researcher → Analyst → Risk Manager → Executor. Risk manager can veto trades.

Advanced (6+ Agents)

Per-asset-class agents (Stocks, Crypto, Options) + Orchestrator + Risk Veto + Executor. For sophisticated multi-strategy operations.

Model Selection Per Role

Orchestrator/Risk: Opus/Claude-3.5 (highest reasoning)
Analysis: Sonnet/Claude-3 (balanced performance/cost)
Monitoring: Haiku/Claude-3-fast (fast, cheap for simple tasks)
