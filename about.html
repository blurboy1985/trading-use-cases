🎯 Purpose

This catalogue documents real-world deployments of OpenClaw AI agents for trading and financial automation. Our goal is to help the community learn from each other's experiences, avoid common pitfalls, and build better trading systems together.

📊 Methodology

How We Collect Use Cases

  1. Community submissions via GitHub, Discord, and direct contributions
  2. Verified deployments require proof of operation (screenshots, logs, or public demo)
  3. Pattern documentation for common setups seen across multiple users
  4. Research from OpenClaw docs, showcase, and community forums

Data Fields Captured

Each use case includes 15+ fields: who, trading style, agent setup, what it does, data sources, broker integration, financial impact, risk controls, skills used, infrastructure, monthly cost, regulatory notes, gotchas/failures, source URL, and verification status.

⚠️ Critical Disclaimers

🚨 NOT Financial Advice

Nothing in this catalogue constitutes financial advice, investment recommendations, or trading signals. All use cases are for educational purposes only. You are solely responsible for your trading decisions.

💸 Financial Risk Warning

Automated trading carries substantial risk of loss. Past performance does not guarantee future results. Never deploy real capital without extensive paper trading (minimum 3 months). Only risk money you can afford to lose.

⚖️ Regulatory Compliance

You are responsible for understanding and complying with securities regulations in your jurisdiction (SEC, MAS, FCA, ESMA, etc.). Managing others' money may require licensing. This is not legal advice.

🤖 AI Limitations

AI agents WILL hallucinate financial data. Always verify critical information against primary sources. Never give an agent withdrawal permissions. Always implement kill switches and human approval gates.

🤝 How to Contribute

Submit a Use Case

  1. Fork the repository on GitHub
  2. Add your use case to data/usecases.json following the schema
  3. Include all 15+ fields with honest, accurate information
  4. Set verified: true only if you're actively running this
  5. Submit a pull request with clear description

Improve Existing Entries

See errors, outdated info, or missing gotchas? Submit a PR with corrections. Include sources when possible.

Report Issues

Found a bug, broken link, or inaccurate information? Open a GitHub issue with details.
